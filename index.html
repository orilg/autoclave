What is it?

Autoclave is a skeleton project (template, as some might call it) for rapid development and deployment of web applications.

Web apps require a lot of boilerplate just to get started. How the app is structured, how it is deployed, how would the different technologies at play will be set up are just some of the annoying questions a developer has to face.

While large scale customer-facing websites still need to be carefully crafted to take scaling and other factors into account, many apps that are written today are intended for in-house use, and are generally more 'quick and dirty', putting an emphasis on rapid prototyping and flexibility. For these apps you want a comfortable skeleton to start with, bringing you as fast as possible to the state of deploying a first version.

Autoclave is exactly that. It gives you a lot of facilities out of the box, and is basically ready for deployment straight away. Autoclave is also optimized to be deployed to a dedicated server (usually a VM, which is a likely case for in-house deployments nowadays), but it can also be easily deployed in other environments.

On the Surface

Flask Powered

Autoclave builds on Flask for the backend code. Flask is awesome, minimalistic, expressive and easy to maintain.

Bootstrap Styling

Autoclave uses bootstrap for styling and CSS generation, making your design clean from the start.

Notifications

Autoclave includes pnotify for easy and slick user notifications, popups, and more.

jQuery

jQuery is built in and ready for use.

Under the Hood

All you need, out of the box

Autoclave is powered by MongoDB for storage, redis for caching and IPC purposes, and Celery backed by RabbitMQ for asynchronous tasks.

Nginx + uWSGI

Autoclave uses a default configuration of nginx, forwarding requests via uWSGI. Static files are served directly from nginx to better utilize your server resources.

Simple Deployment and Debugging

Deployment is done via a Fabric file. It can deploy the entire app plus dependencies to a remote server, or run a tmux split-screen setup for debugging, letting you monitor everything in one glance. It also comes ready with a Vagrant file, enabling you to deploy directly to vagrant.

OpenID

Autoclave includes OpenID authentication out of the box.

Getting Started

  1. Clone the source: 
    $ git clone https://github.com/vmalloc/autoclave
  2. Install dependencies: 
    $ pip install -r autoclave/pip_requirements.txt
  3. Edit autoclave/flask_app/config/app.py to customize your app. You'll probably just want to change APP_NAME at first.
  4. To debug your app, run 
    $ fab debug
  5. To deploy to a server, run
    $ fab deploy -H root@myserver -p password
  6. You can also deploy to a Vagrant local VM: 
    $ fab deploy_vagrant
